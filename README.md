# Drupal Website QA Project

## Overview
This project demonstrates my QA testing approach for a Drupal-based website, focusing on content workflows, modules, user roles and permissions, and overall site functionality.

## Key Areas Tested
- Content creation, review, approval, and publishing workflows
- User roles and permission validation
- Drupal module functionality
- End-to-end site behavior
- UI and navigation testing
- Cross-browser and responsive testing

## Testing Scope

### Content Workflows
Validated transitions between Draft, Review, and Published states across different roles.

### User Roles & Permissions
Tested Admin, Content Editor, Authenticated, and Anonymous roles to ensure proper access control.

### Modules
Verified module functionality and ensured no conflicts or broken features.

### Site Functionality
Tested navigation, forms, content display, and overall user experience across devices and browsers.

## Tools Used
- Manual Testing
- Selenium WebDriver (Java)
- Cucumber
- Jira
- Postman
- Chrome DevTools

## Outcome
Ensured reliable content publishing workflows, secure role-based access, and stable site functionality across environments.
